# Framework de aprendizados (Start/Stop/Continue + A21)

Adaptado do PMOSkills (`fakhruldeen/PMOSkills`) ao contexto de Growth.

## Por pilar — Start / Stop / Continue
Para cada pilar, derive dos dados:
- **CONTINUE** (o que deu certo, manter): top performers, padrões vencedores (ex. "criativo vídeo AD07 = melhor CPL", "catálogo ROI Hunter = ROAS 33").
- **STOP** (o que não deu certo, parar): desperdício, ineficiência (ex. "campanha de Visitas ao Perfil consumiu R$447 sem lead", "tráfego frio ROAS 3,9").
- **START** (o que testar/iniciar): oportunidade derivada do gap (ex. "explorar mais estáticos que entregam bom CPL", "realocar verba de topo para catálogo").

## Cada aprendizado relevante vira uma entrada A21
Campos mínimos:
- **Categoria**: Tráfego / Criativo / Gestão / Cliente / CRM / Tracking.
- **Situação** (fato com número e proveniência): "C:3 com CPL R$22,97, 4–6× os campeões [NEKT]".
- **Causa-raiz** (hipótese): "campanha marcada 'revisar' no nome; segmentação ou criativo desalinhado".
- **Impacto**: Alto / Médio / Baixo.
- **Recomendação** (vira sugestão de Problema na matriz; a Solução é do time).

## Status Verde / Âmbar / Vermelho (A17) nos KPIs
- 🟢 ≥ meta/benchmark · 🟡 80–99% · 🔴 < 80% ou em risco.
- Regra "resultado âncora": se o resultado final (vendas/receita) está acima, um topo de funil abaixo **não** é alerta — é eficiência. Leia pelo resultado.

## Da análise para a matriz Problemas | Soluções (seção 8)
1. Liste os achados de maior impacto por pilar.
2. Mapeie cada um ao **integrante responsável** (ver tabela em `pilares.md`).
3. Escreva o **Problema** (fato + número + causa-raiz curta).
4. Deixe a **Solução/Ação em branco** (slot) — o time preenche na reunião.
5. Um quadro por integrante presente na equipe do cockpit (`projectTeam`).
